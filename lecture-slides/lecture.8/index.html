The Tidyverse in Context

  • Introduced by Hadley Wickham; consolidated as meta‑package (2016)
  • Built on ideas from functional programming and data grammar
  • Emphasizes clarity, reproducibility, workflow consistency

Source:Hadley Wickham

Historical Background

  • Early core packages: ggplot2 (2007), dplyr (2014), tidyr (2014)
  • Tidyverse meta‑package formalized in 2016
  • Influenced by the Grammar of Graphics (Wilkinson) and tidy data
  • Pipe‑driven workflows encourage readable code

Tidyverse “Unified Workflow”

Unifies import → wrangle → visualize → communicate

The piped workflow in the Tidyverse in R utilizes the pipe operator (%>%) to chain together data manipulation and analysis functions in a highly readable and intuitive manner.

Source:Tidyverse

Theoretical Underpinnings

Tidy data (Wickham, 2014): Each variable = column, observation = row, value = cell

Source:R for Data Science

What Makes Data “Tidy”

  • Rectangular tables: data frames / tibbles
  • Columns = variables with consistent types
  • Rows = observational units (country > year > cases > population)
  • Enables consistent verbs across the ecosystem

How the Tidyverse Understands Data

In R, a vectorized operation refers to applying a function or operation to an entire vector, matrix, or array at once, rather than iterating through each element individually using explicit loops. This approach leverages R’s underlying optimized C code for efficiency and often leads to more concise and readable code.

  • Transformations preserve table semantics
  • Vectorized operations across columns/rows
  • Declarative: describe what you want, not how to loop

Tibbles vs Data Frames



Tibbles are a modern reimagining of data frames in R, part of the tidyverse, and offer several key differences, particularly in their printing and subsetting behavior.

Tibbles vs Data Frames

  • Data Frames: By default, data frames print all rows and columns, which can be overwhelming for large datasets.

  • Tibbles: Tibbles have a “refined” print method. They display only the first 10 rows and all columns that fit on the screen. Additionally, they show the data type of each column, a feature borrowed from str() function which shows the ‘structure’ of a dataframe.

Tibbles vs Data Frames

  • Data Frames: When subsetting a single column from a data frame using [ or [[, the result can sometimes “drop” its dimensions and return a vector instead of a single-column data frame.

  • Tibbles: Tibbles are stricter in their subsetting behavior. Subsetting a single column with [ or [[ will always return a tibble, preserving the structure. Tibbles also require exact matching for column names, generating a warning if a non-existent column is accessed.

dplyr: Data Manipulation

  • filter() — subset rows by conditions
  • select() — choose/reorder columns
  • mutate() — derive new columns
  • summarize() — aggregate
  • arrange() — sort rows

Grouped Operations

summarize(.groups = "")

  • “drop_last” (default for summarise()): This removes the last grouping variable from the grouping structure.
  • “drop”: This removes all grouping variables, returning an ungrouped data frame.
  • “keep”: This preserves the original grouping structure, meaning the output will be grouped by all the same variables as the input.
  • “rowwise”: This converts the output to a row-wise grouped data frame, where each row is treated as its own group.

Joining Data

  • left_join(x, y, by=) keep all of x, match from y
  • inner_join() intersection only
  • full_join() union of keys

Reshaping with tidyr

Long data has multiple rows per entity (like a person or country), with different variables and time points stored in separate columns.

  • long format is more efficient for plotting and future-proof for aggregations and calculations.

Wide data has one row per entity with each variable or time point in its own column.

  • Wide format is often easier to read and perform row-level calculations on.

Data Visualization in Context

  • Visualization is integral to the tidyverse pipeline
  • ggplot2 embodies the Grammar of Graphics
  • Layers map variables → aesthetics → geoms

Grammar of Graphics (ggplot2)

  • aes() — aesthetic mappings (x, y, color, size, etc.)
  • geom_ — visual marks (point, bar, line)
  • facet_ — small multiples
  • theme_ — style system

ggplot and Tidy Data

  • ggplot2 expects tidy tibbles
  • Each column can be mapped to a visual property
  • Preprocess with dplyr, plot with ggplot2 in one pipeline
